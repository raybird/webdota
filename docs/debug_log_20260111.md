# 除錯紀錄：玩家攻擊無法命中小兵

**日期**: 2026-01-11
**狀態**: 調查中 (In Progress)
**優先級**: 高 (High)

## 問題描述
玩家施放技能（普通攻擊 Space、Q、W、E）時，畫面上有出現技能效果並建立了攻擊判定框 (Hitbox)，但對敵方小兵**沒有造成任何傷害**，控制台 (Console) 也沒有出現 "Hit!" 的命中訊息。

## 已驗證的事實 (基於 Log 證據)

1.  **判定框建立與位置 (Hitbox Creation & Positioning)**: ✅ **運作正常**
    *   Log 確認 Hitbox 的座標會隨著玩家移動而正確更新 (例如: `pos=(11.0, 1.0, 0.0)`)。
    *   隊伍歸屬設定正確 (`ownerTeam=red`)。

2.  **目標識別 (Target Identification)**: ✅ **運作正常**
    *   `HitboxManager` 的迴圈能正確掃描到 `creep_blue_w` 等實體。
    *   **友軍傷害檢查 (Friendly Fire Check)**: ✅ **通過**。Log 有顯示針對敵方小兵的距離檢查，代表程式碼 `if (entity.team === hitbox.ownerTeam ...)` 邏輯正確，沒有錯誤地過濾掉敵人。

3.  **距離計算 (Distance Calculation)**: ✅ **運作正常**
    *   計算出的距離 (Dist): 約 `1.15` 至 `1.73`
    *   Hitbox 半徑 (Radius): `3.0`
    *   判定門檻 (Limit): `<=` `3.5` (3.0 + 0.5)
    *   數學驗證: `1.23 <= 3.5` 為 **TRUE**。

4.  **異常現象 (The Anomaly)**: ❌ **失敗點**
    *   儘管事實 1-3 都證明應該命中，但 `hits.push(...)` 動作似乎沒有生效，或者後續被忽略。
    *   **缺失證據**: 我們完全沒看到由 `GameEngine` 或 `entity.takeDamage` 發出的 "Hit!" 或受傷與血量變化的 Log。

## 問題範圍縮小

失敗點必定位於以下兩個位置之一：

### A. `HitboxManager.update()` 內部
即使數學判斷 `1.23 <= 3.5` 為真，程式碼區塊 `hits.push(...)` 卻沒有被執行。
*   *可能原因*: 是否有隱藏的 `hitbox.hitTargets.has(entityId)` 檢查回傳了 true？(系統誤以為已經打過這隻小兵了)。
    *   *檢查點*: 新攻擊產生時 `hitbox.hitTargets` 應該要是空的。

### B. `GameEngine.ts` 內部
`HitboxManager` 正確回傳了包含資料的 `hits` 陣列，但 `GameEngine` 沒有正確處理它。
*   *可能原因*:
    *   `creepManager.getAllCreeps().get(hit.targetId)` 回傳了 `undefined`？
    *   `takeDamage` 被呼叫了但 Log 被過濾？(可能性低)。

## 下一步計畫

1.  **隔離 `hits.push`**: 確認 `HitboxManager` 是否真的有把命中資料放進陣列。(已添加 `[Hitbox-Success]` log，待測試)。
2.  **驗證 GameEngine 接收端**: 確認 `GameEngine` 是否收到非空的 `hits` 陣列。
3.  **驗證小兵查找**: 確認在命中當下 `creepManager.getAllCreeps()` 的內容是否正確。
